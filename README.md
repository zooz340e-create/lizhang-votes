# 里長票數計算機

選你家的「里」，馬上算出選里長要拿幾票才能**退回保證金**、看這個里**近三屆實際當選票**，
以及在地**競爭狀況**（現任連任幾屆、是否同額競選）。定位是行銷品牌的免費引流小工具，
走「開票速報／競選海報」視覺風格。

> **線上：https://lizhang-votes.vercel.app**
> 範圍：**全臺 22 縣市，共 7,973 個里**，三層選單（縣市→區→里）。
> 縣市清單由中選會 API 動態取得，資料按縣市拆檔、前端按需載入（首屏只載 4KB 索引）。

## 三張卡怎麼來的

| 卡片 | 內容 | 可信度 |
|---|---|---|
| ② 退保證金門檻 | 選舉人數 × 10%（《選罷法》），無條件進位 | **接近官方**（誤差 < 3.2%，已驗證） |
| ③ 當選要幾票 | **近三屆實際當選票** + 過半參考線 + 「看幾人參選」說明 | 真實歷史，不做假裝很準的單一預測 |
| ④ 你的參選機會 | 參選機會指數 0–100 → 🟢大好機會/🟡有機會/🟠拚拚看/🔴硬仗 | 真實歷史評分 |

### 為什麼當選票數「不給單一預測數字」

回測（站在 2022 選前、只用 2014+2018 資料預測 2022）顯示：競爭型選區只命中 2/9。
里長選舉變數太大，**關鍵是「那年有幾人參選」**——同一個里兩人對決要 2,500 票，
四人混戰可能 1,200 票就上。所以改用真實歷史當標竿，比假裝精準誠實也更有用。

### 已驗證的準度（`node etl/accuracy.mjs`）

人口推估的選舉人數 vs 中選會 2022 官方數字，15 個里誤差全部 **< 3.2%**
（西寶里 -0.1%、員林里 +0.3%）。保證金門檻因此可當「接近官方」的硬數字。

## 參選機會指數（已定版）

哲學：**實戰驗證優先 + 一律正向講「你的機會」**，不對真人下負評（避免炎上）。
分數高 = 你機會大；分數低 = 現任人氣穩固（誇對方）。`competition()` in `calc.ts`：

`參選機會指數 = clamp(50 + 各因子, 0, 100)`，因子（正向敘述）：

| 因子 | 加分（機會大） | 範圍 |
|---|---|---|
| 上屆競爭緊度 | 上屆票數越接近越加分 | −22 ~ +30 |
| 走勢 | 現任票逐屆下滑 → 「新人空間擴大」 | −12 ~ +18 |
| 動員空間 | 投票率越低、越多票沒被動員 | −8 ~ +14 |
| 舞台型態 | 長期同額沒人卡位 +20 / 近期無人挑戰 +6 | 0 ~ +20 |
| 現任年資 | 年長、可能交棒（用出生年推算） | 0 ~ +16 |
| 強敵在場 | 上屆第二名很強（會分票）→ 扣分 | −14 ~ 0 |

分級：🟢大好機會 ≥70 ／ 🟡有機會 50–69 ／ 🟠拚拚看 30–49 ／ 🔴硬仗 <30 ／ ⚪資料不足。
每張卡顯示：**參選機會指數 + 進度條 + 等級**、**登頂（當選）大約要幾票**、連任屆數、
**機會指數加減分明細**、一句正向結論。

> ⚠️ 用詞紀律：禁止「崩/弱/鬆/虛」等貶低真實里長的字眼。走勢下滑要講「新人空間擴大」、
> 年長講「可能交棒」、強現任講「人氣穩固」。真人名要顧及觀感，避免炎上。
>
> 範例：六寶里 指數 88 🟢（上屆險勝＋現任 69 歲可能交棒）；忠義里 指數 26 🔴（現任贏 63%、人氣穩固）。

## 資料來源

- **人口 / 選舉人數**：內政部戶政司「村里戶數、單一年齡人口」（data.gov.tw 77132，
  快照 2020-11；選舉人數 = 20 歲以上設籍人口推估）。
- **歷屆里長得票**：中央選舉委員會選舉資料庫（2014/2018/2022 第 2–4 屆），
  由 `etl/fetch_elections.mjs` 直接抓中選會靜態 JSON（公開、免登入）自動落地。

## 開發

```bash
npm install
npm run dev               # 本機預覽 http://localhost:5173
npm test                  # calc 單元測試（Node 原生，免轉譯）
npm run build             # 型別檢查 + 打包 dist/
node etl/fetch_all.mjs    # 重抓全臺 22 縣市 → public/data/（縣市清單自動取得）
```

部署：`dist/` 為純靜態檔，直接丟 Vercel（與素材大師同套流程）。

## 檔案結構

- `public/data/index.json` — 縣市清單 + meta（首屏只載這個，約 4KB）。
- `public/data/county/<code>.json` — 各縣市完整里資料（選了縣市才載，20KB–840KB）。
- `src/lib/data.ts` — 資料載入層：按需 fetch + 記憶體快取。
- `src/lib/calc.ts` — 三個計算模組（純函式）：`depositThreshold` / `winInsight` / `competition`。
- `src/lib/calc.test.ts` — 6 個單元測試。
- `src/App.tsx` — 單頁 UI（開票速報風）。`src/index.css` — 設計 token（墨藍/競選紅/燙金/報紙米白）。
- `etl/fetch_all.mjs` — 抓全臺（縣市清單動態取得）。`etl/notes.md` — 資料路徑筆記。
